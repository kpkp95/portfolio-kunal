# Project: Domain RAG Assistant

## Overview
The Domain RAG Assistant is an AI-powered chatbot capable of answering questions using domain-specific documents. The project uses Retrieval-Augmented Generation (RAG) architecture with vector search and LLM generation.

The initial version focused on the medical domain, but the system was designed to later support multiple domains such as:
- Medical
- Machine Learning
- LLMs

## Main Features
- PDF ingestion pipeline
- Document chunking
- Embedding generation
- Vector similarity search
- Domain filtering
- AI-generated answers
- Source-aware retrieval
- Cloud deployment
- Docker containerization

## Tech Stack
Frontend:
- HTML
- Bootstrap
- JavaScript

Backend:
- Flask
- LangChain
- Python

AI Stack:
- Gemini API
- HuggingFace Embeddings
- all-MiniLM-L6-v2

Vector Database:
- Pinecone

Cloud:
- AWS EC2
- Docker
- AWS ECR
- GitHub Actions

## Architecture
User Question → Embedding Search → Pinecone Retrieval → LangChain Context Assembly → LLM Response

## Important Implementation Details
- Metadata-based retrieval filtering
- Chunk overlap strategy
- Context-grounded prompts
- Safety-focused prompt engineering
- Environment variable management
- Dockerized deployment

## Challenges Solved
- Deploying AI applications on AWS
- Managing Docker containers
- Handling EC2 deployment issues
- Configuring Pinecone indexes
- Designing scalable RAG pipelines
- Improving retrieval quality
